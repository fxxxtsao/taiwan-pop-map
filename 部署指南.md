# 台灣人口年齡地圖 — 伺服器部署指南

版本：1.0（2026-07-09）
適用：將本專案移植到其他伺服器對外提供服務。功能規格見 `規格書.md`。

---

## 1. 架構總覽

本專案是**純靜態網站**，沒有後端程式、沒有資料庫。伺服器只需要把 `web/` 目錄的檔案原樣送出，所有運算（熱力圖著色、篩選、聚合）都在使用者瀏覽器內以 JavaScript 完成。

```
┌─ 離線（維護者電腦，每年跑一次）──────────────┐
│  政府開放資料 API/CSV ─► scripts/*.js ─► web/*.json │
└──────────────┬──────────────────┘
               │ 複製 web/ 目錄
               ▼
┌─ 伺服器（唯一需要部署的部分）─────────────┐
│  任一靜態檔案伺服器（nginx / Caddy / IIS / 物件儲存）│
│  內容：HTML/CSS/JS 3 檔 + JSON 資料 8 檔，共 8.3 MB  │
└──────────────┬──────────────────┘
               ▼
┌─ 使用者瀏覽器（執行期外部相依）────────────┐
│  ・d3.js / topojson / d3-tile：cdn.jsdelivr.net       │
│  ・Noto Sans TC 字型：fonts.googleapis.com            │
│  ・道路底圖圖磚：wmts.nlsc.gov.tw（國土測繪中心，免金鑰）│
└─────────────────────────────────┘
```

重點：**資料管線（scripts/）不需要部署到伺服器**。它只在年度更新時於任何一台有 Node.js 的電腦執行，產出新 JSON 後重新上傳即可。

## 2. 硬體需求

| 情境 | 規格 |
|------|------|
| 最低（自架 VM） | 1 vCPU / 512 MB RAM / 1 GB 磁碟 — 靜態檔案服務幾乎不耗資源 |
| 建議（自架 VM） | 1 vCPU / 1 GB RAM，頻寬才是瓶頸（見下） |
| 免費託管替代 | GitHub Pages / Cloudflare Pages / Netlify — 不需任何伺服器，把 `web/` 推上去即可 |

**頻寬估算**：首屏約 1.7 MB（HTML/CSS/JS + 鄉鎮界 + 人口 + 學校 + 所得，gzip 後約 0.9 MB）；使用者放大到村里層再 lazy load 6.4 MB（villages-10t.json + population_villages.json，gzip 後約 2.2 MB）。每千次完整瀏覽約 3 GB 流量。CDN/免費託管方案可忽略此項。

## 3. 軟體需求

### 伺服器端（提供服務）
- 任一靜態檔案伺服器：nginx（建議）、Caddy、Apache、IIS，或 S3/R2 類物件儲存 + CDN
- HTTPS 憑證（Let's Encrypt 免費；免費託管方案內建）
- **不需要** Node.js、資料庫、任何 runtime

### 維護端（年度資料更新，非伺服器）
- Node.js ≥ 18（實測 v24；管線用到內建 `fetch`）
- npm 套件：`topojson-client`、`d3-geo`、`commander`（`npm install` 依 package.json 安裝）
- curl（抓政府開放資料）

## 4. 部署步驟

### 方案 A：免費靜態託管（最簡單，建議）

1. 把 `web/` 目錄內容推到 GitHub repo（或直接拖進 Cloudflare Pages / Netlify）。
2. 設定發佈目錄為該目錄根（`index.html` 所在層）。
3. 完成。HTTPS、CDN、gzip 全部自動。

### 方案 B：自架 nginx

1. 安裝 nginx，將 `web/` 整個目錄複製到伺服器，如 `/var/www/popmap/`。
2. 站台設定（重點是 gzip 與快取策略）：

```nginx
server {
    listen 443 ssl;
    server_name map.example.com;
    # ssl_certificate ...（certbot 自動配置）

    root /var/www/popmap;
    index index.html;

    gzip on;
    gzip_types application/json text/css application/javascript;
    gzip_min_length 1k;

    # JSON 資料與帶 ?v= 版本參數的 css/js 可長快取；
    # index.html 不快取，改版時只要遞增 html 內的 ?v= 就能讓使用者拿到新檔
    location ~* \.(json|js|css)$ {
        expires 30d;
        add_header Cache-Control "public";
    }
    location = /index.html {
        add_header Cache-Control "no-cache";
    }
}
```

3. `nginx -t && systemctl reload nginx`，開瀏覽器驗證（見第 6 節檢查清單）。

### 方案 C：Docker

專案根目錄已有 `Dockerfile` + `docker/nginx.conf`（gzip 與快取策略已設好）：

```bash
docker build -t taiwan-pop-map .
docker run -d -p 8080:80 --restart unless-stopped taiwan-pop-map
```

資料年度更新後重新 build + run 即完成部署（image 只含 web/ 約 9MB + nginx:alpine）。

### 方案 D：Windows / IIS

1. 複製 `web/` 到站台目錄，新增站台指向它。
2. IIS 需加 MIME 對應：`.json` → `application/json`（預設沒有，缺了 JSON 會 404）。
3. 啟用靜態內容壓縮。

## 5. 檔案清單（web/，共 8.3 MB）

| 檔案 | 大小 | 載入時機 |
|------|------|---------|
| index.html / style.css / app.js | 37 KB | 首屏 |
| towns-10t.json（鄉鎮界） | 500 KB | 首屏 |
| counties-10t.json（縣市界） | 141 KB | 首屏 |
| population.json（鄉鎮人口） | 163 KB | 首屏 |
| income.json + income_villages.json（綜稅所得） | 285 KB | 首屏 |
| schools.json（學校點位） | 826 KB | 首屏 |
| villages-10t.json（村里界） | 4.1 MB | 放大 ≥6 倍時 lazy load |
| population_villages.json（村里人口） | 2.4 MB | 同上 |

## 6. 部署後檢查清單

1. 首頁載入，台灣地圖著色正常（不是全灰——全灰代表 JSON 沒載到，查 MIME/路徑）。
2. 道路底圖出現（確認伺服器沒有 CSP 擋掉 `wmts.nlsc.gov.tw`）。
3. 滾輪放大到 6 倍以上，村里界出現（驗證兩個大 JSON 的 lazy load）。
4. 切「綜稅所得」主題，地圖重新著色、圖例顯示「中位數（111年）」。
5. 選一個縣市，大頭針落下動畫正常。
6. F12 Network 確認 JSON 有 `Content-Encoding: gzip`。

## 7. 外部相依與離線環境注意事項

以下三項由使用者瀏覽器直連外部服務，伺服器本身不需要對外連線。若部署在**內網/離線環境**則需自理：

| 相依 | 用途 | 離線替代 |
|------|------|---------|
| cdn.jsdelivr.net | d3@7、topojson-client@3、d3-tile@1 | 下載三個 min.js 放進 web/，改 index.html 的 script src |
| fonts.googleapis.com | Noto Sans TC 字型 | 移除 link（退回系統字型）或自架字型檔 |
| wmts.nlsc.gov.tw | 道路底圖圖磚 | 無簡單替代；UI 可關閉道路底圖，功能不受影響 |

其他注意：
- 政府 WMTS 為免金鑰公開服務，但流量禮儀上不建議做成高流量商用站。
- 頁尾授權標註（政府資料開放授權、© OpenStreetMap）依 ODbL 與開放授權條款**必須保留**。
- 全站無任何金鑰、密碼、個資；repo 可公開。

## 8. 年度資料更新（維護端）

在任一台有 Node.js 的電腦執行 `規格書.md` §3.4 的更新流程（人口 → 學校 → 所得 → QA），產出新的 `web/*.json` 後：

1. 遞增 `index.html` 內 css/js 的 `?v=` 版本號（防瀏覽器快取吃舊檔）。
2. 重新上傳整個 `web/` 目錄（方案 A 就是 git push）。

不需要重啟伺服器、沒有停機時間。
