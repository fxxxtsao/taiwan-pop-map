# 台灣人口年齡地圖 — 系統規格書

版本：1.0（2026-06-12）
專案目錄：`taiwan-pop-map/`

---

## 1. 專案概述

互動式網頁地圖，以全台 368 個鄉鎮市區為單位，呈現依年齡層篩選的人口分布熱力圖（choropleth），並可疊加全國國小與幼兒園位置標記。

| 項目 | 內容 |
|------|------|
| 形式 | 純靜態單頁網頁（無後端） |
| 人口資料 | 內政部戶政司，民國 114 年 12 月（2025 年底，最新完整年度）村里級單一年齡人口，聚合至鄉鎮市區 |
| 行政區界 | [dkaoster/taiwan-atlas](https://github.com/dkaoster/taiwan-atlas)（TopoJSON，WGS84 經緯度） |
| 學校名錄 | 教育部統計處：國小 114 學年（2,608 所）、幼兒園 113 學年（6,813 所，114 學年尚未發布） |
| 視覺主題 | 乾淨透明（毛玻璃卡片）、柔和暖色調 |

## 2. 功能需求

### 2.1 人口熱力圖
- **雙層級**：全圖視野以鄉鎮市區著色；縮放 ≥ 6 倍（或選縣市後）自動切換為**村里級**熱力圖（7,701 村里，首次載入時 lazy load 約 6.6MB），鄉鎮界改為白色外框疊在村里層上。可於「底圖與界線」關閉自動切換。
- 色階依「目前選取的年齡層 × 呈現方式」計算，鄉鎮/村里各自獨立計算分位數，圖例隨當前層級切換。
- 年齡層快選：總人口、0–5（學齡前）、6–11（國小）、12–17（國高中）、18–39、40–64、65+。
- 自訂年齡區間：兩個數字輸入框（0–100，100 代表 100 歲以上）。
- 呈現方式：
  - **人口數**：區間內人口加總。
  - **占總人口比例**：區間人口 ÷ **所屬縣市總人口**（鄉鎮與村里層一致；呈現各區/里對縣市的貢獻分布。詳細面板的「占比」仍為區間人口 ÷ 該區總人口，即區內年齡結構）。
- 色階以全體鄉鎮值的 2%–98% 分位數為域（避免極端值壓縮色階），連續漸層，超界截斷。

### 2.2 學校圖層
- 國小（藍）、幼兒園（琥珀）兩個可獨立開關的圖層。
- **未選縣市/鄉鎮**：以固定螢幕尺寸（約 2.2px）小圓點呈現，且縮放倍率 ≥ 3 時才顯示（全圖視野下隱藏，避免雜訊）。
- **選定縣市（下拉選單）或點選鄉鎮**：該範圍內的學校改以「立體大頭針」標記呈現（漸層球形針頭＋金屬針身＋地面陰影），進場動畫為自高處落下→插入地面（微下壓）→回彈，依序錯落出現；範圍外學校維持小圓點規則。
- **同址合併**：國小與其附設幼兒園（或任何同座標的國小＋幼兒園組合）合併為一支**紫色**大頭針，tooltip 同時列出兩所學校並各帶類型標籤；若其中一個圖層被關閉則退回單色針。
- 大頭針為固定螢幕尺寸（針高約 27px），不隨地圖縮放改變。
- 滑鼠懸停（圓點與大頭針皆可）顯示校名、類型、公/私立、所在鄉鎮。

### 2.2-1 道路底圖
- 內政部國土測繪中心臺灣通用電子地圖（EMAP）WMTS 圖磚（免金鑰），鋪於熱力圖之下；熱力圖以 72% 不透明度疊加，道路、市區紋理可透出。
- 圖磚經降飽和處理（saturate 0.6）以維持柔和主題；可於控制面板開關。
- WMTS 端點：`https://wmts.nlsc.gov.tw/wmts/EMAP/default/GoogleMapsCompatible/{z}/{y}/{x}`，以 d3-tile 隨縮放動態換層，圖磚為螢幕座標層（不隨 SVG 變形，保持清晰）。

### 2.3 互動
- 滑鼠滾輪／拖曳：縮放、平移（縮放範圍 1–60 倍）。
- 懸停鄉鎮（或村里層級時的村里）：tooltip 顯示行政區名、選取年齡層數值、總人口。
- 村里層級時點選村里＝選取其所屬鄉鎮（縮放並開啟詳細面板、落大頭針）。
- 點選鄉鎮：放大至該鄉鎮並開啟右側詳細面板。
- 詳細面板：總人口、選取年齡層人數與占比、0–100 歲單齡直方圖（選取區間以強調色標示）、轄內國小／幼兒園數。
- 縣市下拉選單：由北至南排序（基隆→台北→…→屏東→宜花東），離島（澎湖、金門、連江）排最後；選擇後縮放至該縣市並落下大頭針；重設按鈕回全圖。

### 2.4 非功能需求
- 不依賴金鑰或付費服務；所有資料為開放資料。
- 首屏資料量 < 2 MB（TopoJSON 512 KB + 人口 163 KB + 學校約 700 KB + 函式庫 CDN）。
- 桌面瀏覽器（Chrome/Edge/Firefox/Safari 近兩年版本）；行動版基本可用（卡片縮排）。

## 3. 資料來源與管線

```
內政部戶政司 API ─┐
taiwan-atlas ────┼─► scripts/（Node.js 前處理）─► web/*.json ─► 瀏覽器 D3 渲染
教育部統計處名錄 ─┤
OSM Overpass ────┤（座標比對）
OSM Nominatim ───┘（地址地理編碼，補缺）
```

### 3.1 人口（鄉鎮市區 × 單一年齡）
- 來源：戶政司開放資料 API `ODRP014`「村里戶數、單一年齡人口（含新增區域代碼）」
  `https://www.ris.gov.tw/rs-opendata/api/v1/datastore/ODRP014/{yyymm}?PAGE={n}`（yyymm 如 `11412`，共 4 頁，每頁 2,000 筆村里）。
- 處理：`scripts/aggregate_population.js` 將 7,752 村里依 `district_code` 前 8 碼聚合為 368 鄉鎮，輸出 `web/population.json`。
- 鄉鎮代碼與 taiwan-atlas 的 `TOWNCODE` **完全一致**（368/368 對應，已驗證）。

### 3.2 行政區界
- `https://unpkg.com/taiwan-atlas@2021.9.20/towns-10t.json`（鄉鎮）、`counties-10t.json`（縣市界線）。
- 座標為 WGS84 經緯度（非預投影），前端以 `d3.geoMercator().fitExtent()` 投影。

### 3.3 學校名錄與座標
- 名錄（含地址，**無座標**）：
  - 國小：`https://stats.moe.gov.tw/files/school/114/e1_new.csv`
  - 幼兒園：`https://stats.moe.gov.tw/files/school/113/k1_new.csv`
- 全國性「含經緯度」官方資料集不存在（教育部僅於 105 學年公告過一次），故座標以三層策略取得（`scripts/build_schools.js` + `scripts/geocode.js`）：
  1. **OSM 名稱比對**：Overpass 抓取台灣 `amenity=school/kindergarten` 點位，以「縣市 + 正規化校名」比對（正規化：臺→台、國民小學→國小、幼稚園→幼兒園、去除立別與法人前綴）。並以鄉鎮多邊形點包含測試標記每點所屬鄉鎮。全國唯一名稱備援比對**必須同縣市**才採用，防止異縣市同名校（如多縣市皆有的「清華幼兒園」）共用同一點位；匯出時另以縣市多邊形對所有來源座標做落區複驗（`scripts/check_dupes.js` 為同名/共點 QA 工具）。
  2. **附設幼兒園繼承宿主學校座標**：「XX國小附設幼兒園」直接使用 XX 國小座標。
  3. **Nominatim 路名層級地理編碼**：其餘依名錄地址（去除村里鄰、截至門牌號）查詢；台灣門牌在 OSM 幾乎無法解析，實際精度為**路名層級**。仍失敗者於匯出時不含座標（前端不顯示）。
- 匯出 `web/schools.json`，每筆記錄座標來源（`osm`/`host`/`road`）。

### 3.3-1 綜稅所得（村里統計分析表）
- 來源：財政部財政資訊中心「綜稅所得總額各縣市鄉鎮村里統計分析表」，每縣市一個 CSV：
  `https://www.fia.gov.tw/WEB/fia/ias/ias{年度}/{年度}_165-{縣市代碼}.csv`（縣市代碼 A–Z 共 22 碼，如 A=臺北市、G=宜蘭縣）。
- 年度落後約 2–3 年（目前最新 111 年度）；欄位：縣市別（縣市+鄉鎮連寫）、村里、納稅單位(戶)、綜合所得總額、平均數、中位數、Q1、Q3（單位千元）。
- 處理：`scripts/build_income.js` 以「縣市+鄉鎮+村里」名稱比對 taiwan-atlas `VILLCODE`（臺/台正規化＋同鄉鎮一字之差模糊比對救罕見字異體，需以 code point 比較），對應率 99.2%；鄉鎮層以 Σ總額/Σ戶數為平均、村里中位數的戶數加權中位為中位數。輸出 `web/income.json`（鄉鎮）與 `web/income_villages.json`（村里，~275KB 隨首屏載入）。
- 注意：這是綜所稅**申報**資料，不含免稅/分離課稅所得與未申報者；「合計」「其他」列須略過。

### 3.4 更新流程（年度）
```powershell
# 1. 人口（改 yyymm 為最新 12 月）
curl "https://www.ris.gov.tw/rs-opendata/api/v1/datastore/ODRP014/11512?PAGE=1..4" -o data/p{n}.json
node scripts/aggregate_population.js
node scripts/build_villages_pop.js
# 2. 學校名錄（改學年度）
curl https://stats.moe.gov.tw/files/school/115/e1_new.csv -o data/e1_new.csv
curl https://stats.moe.gov.tw/files/school/114/k1_new.csv -o data/k1_new.csv
node scripts/build_schools.js
node scripts/geocode.js        # 增量，有快取
node scripts/export_schools.js
# 3. 綜稅所得（改年度；22 縣市代碼 A B C D E F G H I J K M N O P Q T U V W X Z）
#    每縣市: curl https://www.fia.gov.tw/WEB/fia/ias/ias{年度}/{年度}_165-{代碼}.csv -o data/income/{年度}_{代碼}.csv
node scripts/build_income.js   # 記得改腳本內 YEAR 常數
```

## 4. 系統架構與檔案結構

```
taiwan-pop-map/
├── data/                  # 原始與中間資料（不部署）
│   ├── p1..p4.json        # ODRP014 原始分頁
│   ├── e1_new.csv         # 國小名錄 114 學年
│   ├── k1_new.csv         # 幼兒園名錄 113 學年
│   ├── osm_schools.json   # Overpass 結果
│   ├── geocode_cache.json # Nominatim 快取（斷點續跑）
│   └── schools_stage1.json
├── scripts/               # Node.js 資料管線
│   ├── aggregate_population.js   # 村里 API → 鄉鎮聚合
│   ├── build_villages_pop.js     # 村里 API → 村里級人口
│   ├── build_schools.js          # 名錄 × OSM 比對
│   ├── geocode.js                # Nominatim 地理編碼（可續跑）
│   ├── export_schools.js         # 合併 + 點位落區驗證 + 匯出
│   └── build_income.js           # 綜稅所得 CSV × VILLCODE 比對 + 鄉鎮聚合
├── web/                   # 可直接部署的靜態網站
│   ├── index.html / style.css / app.js
│   ├── towns-10t.json / counties-10t.json / villages-10t.json
│   ├── population.json / population_villages.json
│   ├── income.json / income_villages.json
│   └── schools.json
└── 規格書.md
```

技術選型：MapLibre GL JS v4（WebGL 渲染：底圖、熱力圖、學校點、縮放平移）+ D3.js v7（分位數計算、圖例、直方圖）+ topojson-client（CDN）、原生 JS（無框架、無建置步驟）、Noto Sans TC。大頭針為 HTML Marker（保留立體造型與落下動畫，超過 400 支自動退回圓點）。
本地預覽：`python -m http.server 8123 --directory web`（需經 HTTP 載入 JSON，不能直接開檔案）。

## 5. 資料規格

### 5.1 `population.json`
```jsonc
{
  "source": "內政部戶政司 ODRP014",
  "period": "11412",                  // 民國年月
  "towns": {
    "65000010": {                     // taiwan-atlas TOWNCODE（8 碼）
      "name": "新北市板橋區",
      "total": 545210,
      "ages": [3210, 3300, /* 索引 = 年齡 0..100，100 含以上 */]
    }
  }
}
```

### 5.1-1 `population_villages.json`（村里層，lazy load）
```jsonc
{
  "period": "11412",
  "villages": {
    "65000010001": { "n": "留侯里", "t": 1641, "a": [/* 0..100 單齡 */] }
  }
}
```

### 5.2 `schools.json`
```jsonc
{
  "updated": "2026-06-12",
  "elemYear": 114, "kinderYear": 113,
  "elem":   [["板橋國民小學", 25.01234, 121.45678, "公立", "新北市", "板橋區", "osm"], ...],
  "kinder": [[...]]   // 欄位同上：[校名, 緯度, 經度, 公/私立, 縣市, 鄉鎮, 座標來源]
```

## 7. 資料品質與已知限制

1. **幼兒園名錄年度落後**：教育部 114 學年幼兒園名錄尚未發布，目前使用 113 學年；國小為 114 學年。
2. **座標精度分級**：`osm`（建物級）＞ `host`（宿主國小，建物級）＞ `road`（路名層級，同路多校會落在相近位置）＞ `town`（鄉鎮中心點，為地理編碼跨區錯置之回退值）。匯出時以鄉鎮多邊形做點位落區驗證。
3. **少數學校無座標**：地理編碼全失敗者暫不顯示（匯出時統計會列出缺漏數）。
3-1. **村里界版本差**：taiwan-atlas 村里界為 2021 版，與 114 年人口資料比對有 66 個村里（0.9%）因整併對不上，顯示為灰色。
4. 人口為**戶籍**人口，與實際居住人口有差異（通勤城市低估、戶籍掛留高估）。
5. 100 歲欄位代表「100 歲以上」合計。
6. OSM 衍生座標需依 ODbL 標註「© OpenStreetMap」（頁尾已標註）。
