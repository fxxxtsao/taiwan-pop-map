# 台灣人口年齡地圖 — 系統規格書

版本：1.3（2026-07-24）
專案目錄：`taiwan-pop-map/`
線上版：https://fxxxtsao.github.io/taiwan-pop-map/　｜　原始碼：https://github.com/fxxxtsao/taiwan-pop-map

---

## 1. 專案概述

互動式網頁地圖，以全台 368 個鄉鎮市區（可下鑽至 7,701 村里）為單位，呈現依年齡層篩選的人口分布與綜稅所得熱力圖（choropleth），並可疊加全國國小、幼兒園、補習班、KUMON 教室四個點圖層；國小與幼兒園另以圓點深淺表達學校規模。

| 項目 | 內容 |
|------|------|
| 形式 | 純靜態單頁網頁（無後端） |
| 人口資料 | 內政部戶政司，民國 114 年 12 月（2025 年底，最新完整年度）村里級單一年齡人口，聚合至鄉鎮市區 |
| 所得資料 | 財政部財政資訊中心 111 年度綜稅所得（村里級中位數／平均數） |
| 行政區界 | [dkaoster/taiwan-atlas](https://github.com/dkaoster/taiwan-atlas)（TopoJSON，WGS84 經緯度） |
| 國小 | 教育部統計處 114 學年名錄，2,582 所有座標；併入 114 學年校別學生數／班級數 |
| 幼兒園 | 全國教保資訊網 `is_active=1` 母體 7,311 園（含核定招收人數），非教育部名錄 |
| 補習班 | 教育部全國短期補習班資管系統：文理＋外語 15,602 家 |
| KUMON | 使用者維護的現營業教室清單 178 家 |
| 視覺主題 | 乾淨透明（毛玻璃卡片）、柔和暖色調 |
| 部署 | GitHub Actions → GitHub Pages（推 `main` 自動部署 `web/`）；另附 Dockerfile（nginx:alpine） |

## 2. 功能需求

### 2.1 人口熱力圖
- **雙層級**：全圖視野以鄉鎮市區著色；縮放 ≥ 6 倍（或選縣市後）自動切換為**村里級**熱力圖（7,701 村里，首次載入時 lazy load 約 6.6MB），鄉鎮界改為白色外框疊在村里層上。可於「底圖與界線」關閉自動切換。
- 色階依「目前選取的年齡層 × 呈現方式」計算，鄉鎮/村里各自獨立計算分位數，圖例隨當前層級切換。
- 年齡層快選：總人口、0–5（學齡前）、6–11（國小）、12–17（國高中）、18–39、40–64、65+。
- 自訂年齡區間：兩個數字輸入框（0–100，100 代表 100 歲以上）。
- 呈現方式：
  - **人口數**：區間內人口加總。
  - **占總人口比例**：區間人口 ÷ **所屬縣市總人口**（鄉鎮與村里層一致；呈現各區/里對縣市的貢獻分布。詳細面板的「占比」仍為區間人口 ÷ 該區總人口，即區內年齡結構）。
- 色階以全體鄉鎮值的 2%–98% 分位數為域（避免極端值壓縮色階），連續漸層，超界截斷。

### 2.1-1 綜稅所得主題
- 主題切換（人口年齡／綜稅所得）；選所得時年齡層控制列停用，呈現方式改為中位數／平均數。
- 同樣支援鄉鎮↔村里雙層級與相同的分位數色階邏輯。

### 2.2 點圖層（學校／補習班）
- 四個可獨立開關的圖層：**國小**（深藍）、**幼兒園**（暗金）、**補習班**（綠）、**KUMON**（湖水藍）。配色規範見 6.1。
- 全部僅在縮放 ≥ `SCHOOL_MINZOOM`（8.2，約舊版 3 倍）時顯示，全圖視野下隱藏避免雜訊。補習班與 KUMON 兩列在對應 JSON 載入完成前隱藏，且預設不勾選。
- **規模編碼**：國小圓點深淺 = 在學學生數、幼兒園 = 核定招收人數，採**連續對數色階**（域分別為 20–3,024 與 12–1,200）；無資料者為灰點。控制面板附兩條漸層圖例。補習班與 KUMON 為單色（無規模資料）。
- **選定縣市（下拉選單）或點選鄉鎮**：該範圍內的國小／幼兒園改以「立體大頭針」呈現（漸層球形針頭＋金屬針身＋地面陰影），進場動畫為自高處落下→插入地面（微下壓）→回彈，依序錯落出現；範圍外維持圓點。補習班與 KUMON 永遠是圓點，不落大頭針。
- **同址合併**：國小與其附設幼兒園（或任何同座標的國小＋幼兒園組合）合併為一支**左深藍／右暗金對半大頭針**，tooltip 同時列出兩所學校並各帶類型標籤；若其中一個圖層被關閉則退回單色針。
- 大頭針為固定螢幕尺寸（針高約 27px），不隨地圖縮放改變；同一範圍 ≤ 150 支走 HTML Marker（逐支錯落落下），超過改 GPU symbol layer（貼圖版，全體落下）。
- 滑鼠懸停（圓點與大頭針皆可）顯示校名、類型、公/私立、所在鄉鎮；國小另列學生數、班級數、平均每班人數，幼兒園列核定招收人數，並附規模標籤（小型／中小型／中型／大型／超大型）。

### 2.2-1 道路底圖
- 內政部國土測繪中心臺灣通用電子地圖（EMAP）WMTS 圖磚（免金鑰），鋪於熱力圖之下；熱力圖以 72% 不透明度疊加，道路、市區紋理可透出。
- 圖磚經降飽和處理（saturate 0.6）以維持柔和主題；可於控制面板開關。
- WMTS 端點：`https://wmts.nlsc.gov.tw/wmts/EMAP/default/GoogleMapsCompatible/{z}/{y}/{x}`，以 d3-tile 隨縮放動態換層，圖磚為螢幕座標層（不隨 SVG 變形，保持清晰）。

### 2.3 互動
- 滑鼠滾輪／拖曳：縮放、平移（縮放範圍 1–60 倍）。
- 懸停鄉鎮（或村里層級時的村里）：tooltip 顯示行政區名、選取年齡層數值、總人口。
- 村里層級時點選村里＝選取其所屬鄉鎮（縮放並開啟詳細面板、落大頭針）。
- 點選鄉鎮：放大至該鄉鎮並開啟右側詳細面板。
- 詳細面板：總人口、選取年齡層人數與占比、0–100 歲單齡直方圖（選取區間以強調色標示）、轄內國小／幼兒園數。
- 縣市下拉選單：由北至南排序（基隆→台北→…→屏東→宜花東），離島（澎湖、金門、連江）排最後；選擇後縮放至該縣市並落下大頭針；重設按鈕回全圖。

### 2.4 非功能需求
- **前端**不依賴任何金鑰或付費服務；所有資料為開放資料。（Google Geocoding／Places 僅用於**離線前處理**，金鑰不進版控、不上前端，見 3.5。）
- 首屏約 3.7 MB：towns 501 KB + counties 141 KB + population 163 KB + income 10 KB + income_villages 276 KB + schools 1.0 MB + cram 1.7 MB + kumon 15 KB；村里界 4.1 MB 與村里人口 2.4 MB 為 lazy load（縮放至村里層才抓）。
- 桌面瀏覽器（Chrome/Edge/Firefox/Safari 近兩年版本）；行動版基本可用（卡片縮排）。

## 3. 資料來源與管線

```
內政部戶政司 API ───┐
財政部綜稅所得 CSV ─┤
taiwan-atlas ──────┼─► scripts/（Node.js 前處理）─► web/*.json ─► 瀏覽器 MapLibre 渲染
教育部統計處名錄 ───┤
全國教保資訊網 ─────┤（幼兒園母體＋核定招收數）
補習班資管系統 ─────┤
OSM Overpass ──────┤（校名比對）
OSM Nominatim ─────┤（路名級地理編碼，早期）
Google Geocoding ──┤（門牌級地理編碼，需金鑰）
Google Places ─────┘（以店名補正跨區／非門牌級）
```

### 3.1 人口（鄉鎮市區 × 單一年齡）
- 來源：戶政司開放資料 API `ODRP014`「村里戶數、單一年齡人口（含新增區域代碼）」
  `https://www.ris.gov.tw/rs-opendata/api/v1/datastore/ODRP014/{yyymm}?PAGE={n}`（yyymm 如 `11412`，共 4 頁，每頁 2,000 筆村里）。
- 處理：`scripts/aggregate_population.js` 將 7,752 村里依 `district_code` 前 8 碼聚合為 368 鄉鎮，輸出 `web/population.json`。
- 鄉鎮代碼與 taiwan-atlas 的 `TOWNCODE` **完全一致**（368/368 對應，已驗證）。

### 3.2 行政區界
- `https://unpkg.com/taiwan-atlas@2021.9.20/towns-10t.json`（鄉鎮）、`counties-10t.json`（縣市界線）。
- 座標為 WGS84 經緯度（非預投影），前端以 `d3.geoMercator().fitExtent()` 投影。

### 3.3 學校名錄與座標
- 名錄（含地址，**無座標**）：
  - 國小：`https://stats.moe.gov.tw/files/school/114/e1_new.csv`
  - 幼兒園：`https://stats.moe.gov.tw/files/school/113/k1_new.csv`（僅作為座標繼承來源，母體已改用教保網，見 3.3-4）
- 全國性「含經緯度」官方資料集不存在（教育部僅於 105 學年公告過一次），故座標以三層策略取得（`scripts/build_schools.js` + `scripts/geocode.js`）：
  1. **OSM 名稱比對**：Overpass 抓取台灣 `amenity=school/kindergarten` 點位，以「縣市 + 正規化校名」比對（正規化：臺→台、國民小學→國小、幼稚園→幼兒園、去除立別與法人前綴）。並以鄉鎮多邊形點包含測試標記每點所屬鄉鎮。全國唯一名稱備援比對**必須同縣市**才採用，防止異縣市同名校（如多縣市皆有的「清華幼兒園」）共用同一點位；匯出時另以縣市多邊形對所有來源座標做落區複驗（`scripts/check_dupes.js` 為同名/共點 QA 工具）。
  2. **附設幼兒園繼承宿主學校座標**：「XX國小附設幼兒園」直接使用 XX 國小座標。
  3. **Nominatim 路名層級地理編碼**：其餘依名錄地址（去除村里鄰、截至門牌號）查詢；台灣門牌在 OSM 幾乎無法解析，實際精度為**路名層級**。仍失敗者於匯出時不含座標（前端不顯示）。
- 匯出 `web/schools.json`，每筆記錄座標來源（`osm`/`host`/`road`）。

### 3.3-1 綜稅所得（村里統計分析表）
- 來源：財政部財政資訊中心「綜稅所得總額各縣市鄉鎮村里統計分析表」，每縣市一個 CSV：
  `https://www.fia.gov.tw/WEB/fia/ias/ias{年度}/{年度}_165-{縣市代碼}.csv`（縣市代碼 A–Z 共 22 碼，如 A=臺北市、G=宜蘭縣）。
- 年度落後約 2–3 年（目前最新 111 年度）；欄位：縣市別（縣市+鄉鎮連寫）、村里、納稅單位(戶)、綜合所得總額、平均數、中位數、Q1、Q3（單位千元）。
- 處理：`scripts/build_income.js` 以「縣市+鄉鎮+村里」名稱比對 taiwan-atlas `VILLCODE`（臺/台正規化＋同鄉鎮一字之差模糊比對救罕見字異體，需以 code point 比較），對應率 99.2%；鄉鎮層以 Σ總額/Σ戶數為平均、村里中位數的戶數加權中位為中位數。輸出 `web/income.json`（鄉鎮）與 `web/income_villages.json`（村里，~275KB 隨首屏載入）。
- 注意：這是綜所稅**申報**資料，不含免稅/分離課稅所得與未申報者；「合計」「其他」列須略過。

### 3.3-2 補習班（文理／外語）
- 來源：教育部全國短期補習班資訊管理系統
  `https://bsb.kh.edu.tw/afterschool/opendata/afterschool_json.jsp?city={代碼}`
  **必須 `curl -L`**（301 轉址到 `af.bsb.kh.edu.tw`）。22 縣市代碼：`24 20 21 33 35 36 37 42 47 55 49 52 53 62 70 87 39 38 89 69 82 83`，逐縣市存成 `data/cram/city_{代碼}.json`。
- 全國約 1.7 萬家，篩文理＋外語後 15,602 家。資料自帶 8 碼行政區代碼（95% 直接對上 `TOWNCODE`，舊代碼以地址區名回查修正）。
- 管線：`build_cram.js [縣市名]` → `geocode_cram.js`／`geocode_google.js` → `export_cram.js` → `web/cram.json`。
- 座標為 **Google 門牌級**（見 3.5）。門牌唯一座標率 93%；最大共點 19 家為同棟大樓不同樓層，屬真實情況非錯誤。
- 課後照顧班無全國性資料集：國小自辦的可繼承學校座標，私立中心僅個別縣市開放。

### 3.3-3 KUMON 教室
- 來源：使用者自行維護的 `KUMON_classroom.csv`（現營業教室），欄位含 `center_name`、地址；行政區欄位**不可信**（出現過「沙鹿區鎮」、香山/東區錯置），一律從地址解析。
- 管線：`build_kumon.js`（去重、剔除 `[NULL]` 地址）→ `geocode_kumon.js`（Google Geocoding，共用快取）→ `fix_kumon_places.js`（對非門牌級或落區失敗者以**店名** Places Text Search 補正，只在補正結果通過落區驗證時才寫回）→ `export_kumon.js` → `web/kumon.json`。
- 178 家全數門牌級，落區不符 0。tooltip 顯示 CSV 的 `center_name`。

### 3.3-4 幼兒園母體與規模資料
- **幼兒園母體改用全國教保資訊網**（g0v 鏡像）`https://kiang.github.io/ap.ece.moe.edu.tw/preschools.json`，取 `is_active=1` 共 7,311 園，帶 `count_approved`（**核定**招收人數）。
  - 為何不用教育部名錄：兩者園數差約 817，其中 319 為已停辦（`is_active=0`），其餘為新設立、名稱變更、分班獨立計算等。教保網較即時且附招收數。
  - **全國校別「實際」招收人數未公開**，只有核定數；tooltip 與圖例均標示為「核定招收」。
  - 座標處理：`build_kinder_universe.js` 先以「縣市+鄉鎮+校名」與「縣市+鄉鎮+正規化地址」向既有管線繼承座標（6,423 筆），其餘以 Google 補（house 698、place 174、town 16），最終 0 缺漏。
- **國小學生數／班級數**：教育部統計處校別資料 `https://stats.moe.gov.tw/files/detail/114/114_basec.csv`（`_basej` 國中、`_base0` 高中、`_bases` 補習學校）。`build_elem_students.js` 以**表頭 regex** 定位欄位（`/年級班級數$/`、`/年級[男女]學生數$/`）再加總，避免年度間欄位位移；2,663 校，以學校代碼 100% 併入 `schools.json`。

### 3.4 Google 地理編碼（門牌級）
- 台灣門牌在 OSM／Nominatim 幾乎解析不了，需門牌級精度時一律走 Google：
  - Geocoding `https://maps.googleapis.com/maps/api/geocode/json?region=tw&language=zh-TW&address=…&key=…`
  - Places Text Search `…/place/textsearch/json?language=zh-TW&region=tw&query={店名}&key=…`（地址查不到或跨區時，用店名反查實際店址）
- 金鑰放專案根 `.google_key`，**已 gitignore，不得 commit 或顯示**。快取 `data/google_cache.json`（key = 清理後完整地址，跨管線共用，可斷點續跑）。用量落在每月免費額度內。
- ⚠️ **移機或改目錄後金鑰必須重新建立**——`.google_key` 不在版控內，clone 下來不會有。
- 三個必踩的坑：
  1. `partial_match: true` 是**假精度旗標**，即使 `location_type: ROOFTOP` 也代表 Google 沒真的比到門牌，必須視為失敗。
  2. 段號要**轉中文數字**（`4段155號` → `四段155號`），否則 Google 會誤解析成「三段4號」。
  3. Node `fetch` 遇 `ECONNRESET` 會拋未捕捉例外，必須 try/catch＋重試。
- 所有匯出都做**落區驗證**（`geoContains` 對鄉鎮多邊形），不符者退回鄉鎮中心點。行政區名要拿地址前綴去比對 topo 的真實區名（**取最長匹配**）——用 `(.+?[市區鄉鎮])` 這種非貪婪 regex 會把「平鎮區」截成「平鎮」。

### 6.1 圖層配色（2026-07-24 定案）

點圖層都疊在**低飽和暖色熱力圖**上（`#f5f2ea → #a85f68`，色相集中 15–40°、飽和度 ≤50%、明度 55–96%），配色的唯一目標是讓點在整條底圖色階上都站得住。

| 圖層 | 大頭針 | 圓點／色階 | 描邊 |
|------|--------|-----------|------|
| 國小 | `#285a8c`（亮 `#86b5db`／暗 `#14375a`，頸環 `#1d4771`） | 深藍階 `#c5dcef #93bcdd #5f93c3 #3a6da1 #1f4a76`（學生數） | 白 0.5 |
| 幼兒園 | `#c98a00`（亮 `#ffd98a`／暗 `#6b4200`，頸環 `#6b4a00`） | 暗金階 `#ffd98a #f0b429 #d99400 #b07400 #7a5200`（核定招收數） | 深褐 `rgba(61,43,10,.6)` 0.7 |
| 國小＋附幼同址 | 左深藍／右暗金**對半頭**，白分隔線 1.5，半徑 8.2（單色針 7.2） | — | — |
| 補習班 | — | 綠 `#5aa23a` 單色 | 白 1 |
| KUMON | — | 湖水藍 `#7DCDF4` 單色 | 白 1.2 |

**加新圖層時依序檢查：**

1. **與底圖的距離**——暖色（色相 15–55°）要壓明度、拉飽和才跳得出來；冷色（藍/紫/綠）天生就分得開。
2. **與既有圖層的距離**——目前佔用：深藍(210°)、暗金(41°)、綠(100°)、湖水藍(197°)。新圖層優先取青綠 160° 或洋紅 320° 一帶。
3. **描邊要跟著色相走**——冷色點用白邊；暖色點的白邊會融進米白底圖，必須改深褐或深灰。這是幼兒園改黃時踩到的坑。
4. **同一類實體的大頭針與圓點用同色相**，否則使用者要記兩套對照表。

**規模編碼**用**連續對數色階**（`interpolate` + `log10`），不要用離散分位數 `step`——學生數/招收數是重尾分布，離散分箱會讓尾端全部同色（曾出現 250 與 810 同色）。大頭針只作類型辨識、**不帶規模**（使用者明確要求）。

**沿革與被否決的方案**（避免重複試錯）：

| 曾用配色 | 否決原因 |
|---------|---------|
| 幼兒園圓點琥珀 `#e0a458` | 與暖色底圖幾乎同色相，直接融掉 |
| 幼兒園圓點紫 `#a06fcf` | 可行，且對比最高；後因使用者偏好黃色改掉，要退回改 `KIN_RAMP` 與 `PIN_HEAD.kinder` 即可 |
| 補習班圓點湖水綠 `#4f9d8b` → 深綠 `#2f7565` | 與當時的綠色國小圓點混淆 |
| 國小圓點草綠 `#5aa23a` | 綠色讓給補習班後改深藍，國小大頭針與圓點才統一色相 |
| 同址大頭針單一紫 `#8f7bb5` | 與幼兒園紫圓點撞色，會被誤讀成幼兒園 |

**對半大頭針的實作細節**：左右兩半必須共用同一道球面光，所以漸層改 `gradientUnits="userSpaceOnUse"`（依 r=8.2 換算 `cx=-2.46 cy=-25.28 r=12.3`）；用 `objectBoundingBox` 的話兩個半圓各自算 bbox，高光會錯位。分隔線是獨立畫的 `<line>`，不是右半圓的 stroke——用 stroke 會讓整顆輪廓左右粗細不對稱。HTML Marker 與 GPU 貼圖（縣市級大量大頭針）兩條算繪路徑要同步改。

## 7. 資料品質與已知限制

1. **幼兒園母體與名錄不一致**：已改用教保網 `is_active=1`（7,311 園），與教育部 113 學年名錄（6,813 園）差約 817，成因見 3.3-4。`schools.json` 的 `kinderYear: 113` 僅為歷史欄位，實際是教保網當次抓取快照。
2. **幼兒園是「核定」招收人數，不是實際招收**：全國校別實際招收數未公開。核定數是上限，都會區熱門園所常滿招、偏鄉常遠低於核定，跨區比較要留意。
3. **座標精度分級**：`house`／`place`（門牌級，Google）＞ `osm`／`host`（建物級）＞ `inherit`（沿用先前管線，精度同其原始來源）＞ `road`（路名層級，同路多校落在相近位置）＞ `town`（鄉鎮中心點回退）。所有匯出都以鄉鎮多邊形做落區驗證。
4. **補習班同址堆疊**：門牌唯一座標率 93%，最大共點 19 家（許昌街 17 號同棟不同樓）為真實情況；地圖上會重疊，目前不做聚合，後續可評估 cluster。
5. **少數學校無座標**：地理編碼全失敗者不顯示（匯出時統計列出缺漏數）；幼兒園母體重建後為 0。
6. **村里界版本差**：taiwan-atlas 村里界為 2021 版，與 114 年人口資料比對有 66 個村里（0.9%）因整併對不上，顯示為灰色。
7. 人口為**戶籍**人口，與實際居住人口有差異（通勤城市低估、戶籍掛留高估）。
8. 100 歲欄位代表「100 歲以上」合計。
9. 綜稅所得年度落後 2–3 年（目前 111 年度），與 114 年人口資料**非同年**，交叉解讀要注意。
10. OSM 衍生座標需依 ODbL 標註「© OpenStreetMap」（頁尾已標註）。
